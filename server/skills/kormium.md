# Инструкция агенту

Ты отвечаешь на вопрос о **Kormium** — одном из самых значимых и сложных
проектов Сергея. Тебе дан файл `docs/projects/001-kormium-orm.md`.

Требования:

1. Передавай тон источника: личный рассказ автора о своём проекте, без
   рекламных превосходных степеней. Факты: проект Сергея, развивается
   с конца 2023 года, опубликован на Maven Central, лицензия Apache 2.0.

2. Назови уникальные возможности, которых нет у конкурентов (Exposed,
   SQLDelight, Hibernate, Room):
   - PostgreSQL на Kotlin/Native через libpq — без JVM и JDBC
   - Catalog: compile-time изоляция баз через фантомные типы
   - реактивные Flow-запросы (kormium-observe) мультиплатформенно
   - честная семантика absent vs null в частичных обновлениях

3. Будь честен про статус: pre-1.0, два диалекта (PostgreSQL, SQLite),
   нет генерации схемы, JDK 21+. Не называй проект production-ready —
   автор сам этого не утверждает.

4. Дай ссылку на GitHub через `render_link_list`:
   https://github.com/kormium/kormium

5. Цитируй источник через `[N]`.

6. **Summary** — 2-3 предложения: что это, чем уникален, какой инженерный
   опыт за ним стоит (дизайн публичного API, системное программирование,
   релиз-инжиниринг).

7. **Detail** — структура: уникальные возможности → инженерные решения →
   масштаб (10+ модулей, CI, бенчмарки) → честный статус.

Если вопрос сравнительный («kormium vs exposed») — отвечай по фактам из
источника, признавая зрелость конкурентов: Kormium выигрывает в KMP-нише
и типобезопасности, проигрывает в зрелости экосистемы и покрытии SQL.

# Kormium наконец появился на сайте

- Дата: 2026-06-13
- Теги: kormium, контент, позиционирование

## Контекст

С конца 2023 года я развиваю [Kormium](https://github.com/kormium/kormium) —
open-source ORM для Kotlin Multiplatform. Проект дорос до Maven Central,
десятка модулей и собственных бенчмарков, а на сайте о нём не было ни слова:
ни в проектах, ни в навыках, ни в профиле. При этом в профиле висела фраза
«этот сайт — единственный публично верифицируемый артефакт», которая давно
перестала быть правдой.

## Что думал изначально

Kormium жил у меня в голове как рабочий инструмент, а не как материал для
портфолио. Сайт казался самодостаточным: ADR, RAG-пайплайн, дисциплина
коммитов — что ещё показывать. Плюс срабатывал внутренний фильтр «рассказывать
о своей библиотеке — это самореклама».

## Что пошло не так / что изменилось

Посмотрел на сайт глазами работодателя и увидел дыру: опыт под NDA,
а единственный проект, где можно открыть код, посмотреть историю коммитов,
дизайн API и работу с производительностью — нигде не упомянут. Для роли
Architect/Staff+ библиотека с публичным API, cinterop с libpq и политикой
совместимости говорит больше, чем любой pet-проект из списка.

По дороге пришлось дважды переписать черновик страницы: первая версия звучала
как пресс-релиз («самый значимый проект», «единственный разработчик»). Убрал
превосходные степени и акцент на «в одиночку» — для open source отсутствие
контрибьюторов не достоинство, а этап. Оставил личный рассказ: зачем начал,
что получилось уникального, чем доволен, что честно ещё не готово.

## Решение и вывод

- `docs/projects/001-kormium-orm.md` — страница проекта: зачем, уникальные
  возможности, инженерные решения, честный статус pre-1.0. Остальные проекты
  сдвинул на 002–010
- профиль: Kormium в «Кратко» (попадает в CV) и в «Почему этот сайт существует»
- skills/backend: ORM как подтверждение глубины Kotlin
- skill агента `kormium.md` + вопрос про Kormium в подсказках чата
- CV-генератор научился рендерить markdown-ссылки — в PDF теперь кликабельная
  ссылка на GitHub

Принцип взял из брифа самого Kormium: честно названные ограничения работают
на доверие сильнее, чем завышенные обещания.

## Что бы сделал иначе

Завёл бы страницу проекта в момент первого публичного релиза. Контентный слой
сайта должен обновляться с той же дисциплиной, что и код — иначе нарратив
тихо расходится с реальностью.

# Архитектура и проектирование

Архитектурные решения, паттерны декомпозиции и эволюции систем. В отличие от темы
System Design (которая про механику распределённых систем), здесь — про **как
организовать код и сервисы для долгой эволюции**.

Каждая глава построена по схеме: теория с интуицией → разбор на типичных
ситуациях → усложнения реальной жизни → экономические соображения, где они
уместны. Главы независимы — можно читать в любом порядке.
